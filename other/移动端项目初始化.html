目录结构

主要分为9大模块 分别是 登录注册、首页、分类、搜索、商品详情、购物车/需求清单、支付/确认支付、个人中心、订单

每一个模块可以独立打包发布 以HOME为例 我们看看里面的内容

在我们的目录中 主要分为

  • build(webpack打包用)
  • config(开打打包的基础参数配置)
  • dist (发布到生产环境使用的文件夹)
  • src (项目核心代码文件夹)
  • static (扩展文件 备用)
  • test(测试代码)

如何把项目初始化

准备工作

安装node版本大于4.0.0 和npm 版本大于3.0.0 地址 https://nodejs.org/en/ (一般来说安装node的时候会把npm一起安装)

如果npm版本过低 请使用 npm -g install npm 升级npm

安装淘宝的npm镜像

由于在国内访问npm比较慢 建议使用淘宝的cnpm

安装方法

npm -g install cnpm

项目初始化

首先打开shell或者cmd 进入到项目的文件夹 比如 home

cd home
cnpm install

如图所示

这个动作是指 把所有项目所依赖的js库拉取到本地 使用

启动项目开发环境

npm run dev

提示

DONE Compiled successfully in 4412ms

Listening at http://localhost:8080

启动成功

如果8080端口被占用 导致启动失败 请在 config 文件下面的index.js 26行修改端口号

打包发布

npm run build

打包后的文件不可以直接打开查看 应该传到相应的git服务器上

注意

如果是 home模块打包的时候确保 config 里面 index.js中 40 行

assetsPublicPath: 'https://m.111.com.cn/yyw/wap/home/',

如果是 pay模块打包的时候确保 config 里面 index.js中 40 行

assetsPublicPath: 'https://m.111.com.cn/yyw/wap/pay/',

VUE （框架）

项目中用到的主要框架是vuejs 2.0

Vue.js 通过尽可能简单的 API 实现响应的数据绑定和组合的视图组件。

vue router可以帮我们做单页应用的时候 管理界面分发

vue resource 提供了http ajax jsonp 的请求

vue git 地址：https://github.com/vuejs/vue

vue api 地址：http://cn.vuejs.org/v2/guide/

vue router git 地址 ：https://github.com/vuejs/vue-router

vue router api 地址 ：http://router.vuejs.org/zh-cn/

vue resource git 地址：https://github.com/pagekit/vue-resource

vue resource api 地址：https://github.com/pagekit/vue-resource/blob/develop/docs/http.md
